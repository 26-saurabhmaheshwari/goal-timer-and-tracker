## Tabs / navigation history
- Tab diet (2026-07): no standalone Audit or FAQ tab — Audit (`#viewAudit`) is a card below `#viewLog`, shows whenever `v==='log'` (rendered by `renderLog()` → `renderAudit()`); FAQ is the `setFaq` section inside Settings sidebar (`#setSide` button `data-s="faq"`). `setView()` remaps stale `LS_VIEW` values `'audit'→'log'`, `'faq'→'settings'`.
- **Pot folded into Overview** (nav 7→6): no `viewPot`. Overview sub-tabs via `ovTab` (`'year'|'monthly'|'pot'`): `renderTabs()` builds `#tabs`, `setOv(o)` shows `#ovYear`/`#ovMonthly`/`#ovPot`; `'pot'` calls `renderPot()`+`renderJourney()`. `#ovPot` holds `#potBox` + `#journeyBox` (same IDs so PDF export finds them; export views array uses `'ovPot'`). Legacy `setView('pot')` redirects to overview+pot tab. Valid views: `overview/log/goals/planning/reports/settings`.

## Export surfaces (consolidated #38) — exactly three
1. Reports→Export PDF (`repPdfBtn`) — the only whole-dashboard print entry (old topbar PDF button deleted).
2. Period-modal footer: four inline `.btn.ghost` — `#pePrint` Paper sheet / `#pePoster` Wall poster / `#peIcs` Reminder / `#peWitness` Witness note — plus `#peSave`; each wired with a plain `.onclick`. (A dropdown was tried and reverted — user prefers flat buttons.)
3. Settings→Backup ("⬇ Download backup"/"⬆ Restore backup", ids `expBtn`/`impBtn`).
`window.print()` calls inside generated print-window HTML (paper sheet, poster) are part of those flows, not separate surfaces.

## Goal features (2026-07)
- **Per-goal log** = `goal.logs[]` (rides `LS_GOALS`, no new key). Row `{ym, period, amt, total}`, `total` = cumulative AFTER that addition. Never purged. Display: `📜 Log (n)` button → `goalLogShow(id)` → `#logModal`, renders `prev +amt → total unit` newest first.
- **Mark complete** = `goal.done`. Toggle `goalToggleDone(id)`; button in edit-modal footer (`#gDone`), not on card. Done goals: hidden from section filters, shown in `#secCompleted`, excluded from stats, `paceLabel` 'Done', `.gcard-done`, `+` hidden. Time retained; reopen sets `done=false`.
- **Soft-delete + Trash** = `goal.deleted` + `goal.deletedAt`. `goalDel` soft-deletes (keeps time entries); `goalRestore` clears; `goalPurge` hard-deletes goal + its `type:'goal'` entries. Trash renders in `#secTrashWrap`/`#secTrash` via `renderTrash()` (hidden when empty).
- **Periods as life currency** = pure display. Life bar chip `round(leftDays/10)`; goal det row "≈ N periods to finish" via `goalPeriodsToFinish(x)` (observed pace projected; null → hidden).
- **Next-period pledge** = `goal.commit={ym,period,amt}` (rides `LS_GOALS`). `savePeriodEntry()` → `openCommitStep(ym,period)` → `#commitModal`, one input per active goal for NEXT period (`nextPeriod`). `saveCommits()` writes/clears. Surfaces: `renderPeriodEntry` goal row (🎯 + ✅/⚠️ actual-vs-pledge) and printed paper sheet. One commit per goal (latest wins).
- **Wall poster + period verdict** = `periodVerdict(ym,period[,ents,perDay])` scores a 10-day block vs the fixed-habit plan (`Σ active-habit m/day × days`): `'hit'`≥80% / `'partial'`≥40% / `'miss'` / `'future'` / `'none'`. All entry types count. Uses CURRENT habits for past periods (no habit history — accepted). `printYearPoster()` = new window, 12×3 grid, elapsed pre-coloured, future blank to hand-colour. `#pePoster` in period modal. Save flash appends the verdict colour.
- **Letter to future self** = `LS_LETTER='ptd_letter_v1'` `{text, writtenYm}`; SYNC_FIELDS `['letter', LS_LETTER, loadLetter, 'replace']`. Written in Settings→Personal (`#setLetterSave`). Shows on Overview `#letterCard` via `maybeShowLetter(nudge)` when letter exists AND not dismissed this year AND (nudge = `mostGoalsBehind()` >half behind, OR Dec 31+). Dismiss = device-local `LETTER_SEEN='ptd_letterseen_v1'` (year) — key deliberately NOT `LS_`-prefixed so test.js ignores it. One dismiss hides for the whole year — accepted.
- **Witness note** = pure display. `periodWitness(ym,period)` = 3-line text: verdict / what got done (goal.logs matching, top 4, skips `via` rows) / next-period pledges (top 4). `copyWitness` → clipboard, falls back to dropping text in `msgEl` (clipboard blocked on `file://`). `#peWitness` in period modal.
- **Renounced list** = `LS_RENOUNCED='ptd_renounced_v1'` array of `{id,text}`; SYNC_FIELDS `'id'` merge. `renAdd`/`renDel`/`renderRenounced` → `#renList`. `🚫 Renounced` card at bottom of Goals page (`#renForm`/`#renInput`), rendered in `renderGoals()`.
- **Journey visualization** (generalized, was "Journey to Shiva") = pure display. `renderJourney()` fills `#journeyBox` (Overview→Pot) with one SVG scene per active targeted goal — `journeyGoals()`; NO hardcoded category or name-match. `sjScene(g,gi)`: row of walls (`sjStep(target)` ~≤140), dark right → cleared-green left, walker moves toward a snow-summit-with-flag marker. `journeyScenesHTML()` shared with PDF export. All personal pieces (Shiva/Shivling/mala/Himalaya, `/ons/i` match, `SJ_BIG_CAT`) removed. Done goals excluded via `!x.done`.
- **Feature toggles** = `LS_FEATURES='ptd_features_v1'` `{screen:true, planning:true}`; SYNC_FIELDS `'replace'`. Checkboxes in Settings→Personal (`#setFeatScreen`/`#setFeatPlanning` → `savePersonal` → `saveFeatures`). `applyFeatures()` (startup + save) hides UI only: screen off hides 📱 typetab + `#scrWrap`; planning off hides the nav button, `setView('planning')` redirects. Cloud pull/import reloads the page, so startup call covers restores.
- **Pot = orb + stones jar (#39)**: jar SVG, 36 stones (one per 10-day period), chronological bottom-up 6/row, coloured by `periodVerdict` (green/amber/red/grey; future = dashed outline). Caption placed/36 + counts. Orb + category legend unchanged. Same `#potBox` id.
- **Reports (#40)**: `renderReports()` renders ONE full-width stacked bar chart first (`#repMain`, dataset per category, `colorFor(cat)`, stacked axes, legend bottom) from `repSubjSamples`. Old subject checkboxes = "Drill down · N selected" `repdrop` drawer; `repSel` defaults EMPTY; each ticked subject adds a per-subject bar card below. Don't reintroduce default-selected subjects.

## Convert mini ↔ top-level (#42)
"Parent goal" `<select>` (`#gParent` in `#gParentWrap`) in the goal EDIT modal. Blank detaches (`parentId=null`); picking a parent makes it a mini (inherits+locks cat/unit/minPerUnit). Candidates = non-deleted, non-done, top-level, not itself. Hidden when the goal already HAS minis (one-level rule). Applied on submit only when `goalEdit && gParentWrap` visible. `gParent.onchange` disables `gCat`/`gUnit`.

## 4-tier restructure (2026-07-18)
Live Nuggets (`sec='active'`, cap 5) / Quarterly (`'progress'`, cap 11) / Yearly (`'yearly'`, cap 21) / Someday (fallback). No data migration — internal values kept, only labels changed + `yearly` added. `SEC_CAP`/`SEC_LAB`/`secOfGoal`/`secCount`/`secCapMsg` helpers; cap hard-blocks in goalForm submit (minis exempt). Per-category quota = `LS_QUOTA='ptd_quota_v1'` array `{cat,n}`, SYNC_FIELDS `'replace'`, editor in Settings→Personal (`#quotaRows`/`#quotaAdd`/`#quotaSave`), soft-warn only (`nuggetQuotaHTML` on the Live Nuggets card). PDF export appends Quarterly + Yearly sub-sections.
